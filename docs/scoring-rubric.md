# AI 技能含金量評分 Rubric

本專案的分數代表「這個 AI 技能是否值得一般知識工作者優先投入時間學習」。Agent 只能產生候選評分，正式分數必須經人工審閱後才合併。

## 權重

| 維度 | 權重 | 方向 |
| --- | ---: | --- |
| 實用度 `practicality` | 8 | 分數越高，越能直接節省時間或提升產出 |
| 落地門檻 `accessibility` | 5 | 分數越高，越容易上手與取得 |
| 成熟度 `maturity` | 4 | 分數越高，越接近正式穩定可用 |
| 獨特性 `uniqueness` | 3 | 分數越高，越有明確差異化價值 |

總分公式：

```text
score = practicality * 8 + accessibility * 5 + maturity * 4 + uniqueness * 3
```

等級：

| 等級 | 門檻 |
| --- | ---: |
| A 乾貨 | 80 |
| B 不錯 | 65 |
| C 普通 | 50 |
| 退件 | 0 |

## 1-5 分錨點

### 實用度

| 分數 | 標準 |
| ---: | --- |
| 5 | 可以直接完成高頻工作流，且節省大量時間 |
| 4 | 對明確族群或常見任務有穩定價值 |
| 3 | 有用，但情境較窄或需要人工補強 |
| 2 | 偏展示、開發者實驗或價值不穩 |
| 1 | 目前難以完成實際工作 |

### 落地門檻

| 分數 | 標準 |
| ---: | --- |
| 5 | 一般使用者可直接在既有產品中使用 |
| 4 | 需要付費方案、開啟設定或少量學習 |
| 3 | 需要特定環境、權限或操作流程 |
| 2 | 主要面向工程師、管理員或企業導入 |
| 1 | 需要等待名單、私測資格或高度客製 |

### 成熟度

| 分數 | 標準 |
| ---: | --- |
| 5 | 正式 GA，文件完整且多數目標用戶可用 |
| 4 | Beta 或已廣泛開放，限制清楚 |
| 3 | Public preview、限定方案或區域開放 |
| 2 | Research preview、早期測試或等待名單 |
| 1 | 僅公告、不可用、已下線或資訊不足 |

### 獨特性

| 分數 | 標準 |
| ---: | --- |
| 5 | 明顯改變工作流，競品難以替代 |
| 4 | 有清楚差異化或整合優勢 |
| 3 | 有價值但多家廠商已有類似能力 |
| 2 | 主要是既有功能包裝 |
| 1 | 差異化不足或不可驗證 |

## Evidence Level

| 等級 | 說明 |
| --- | --- |
| `official-doc` | 官方文件、API 文件、產品說明 |
| `official-blog` | 官方 blog、release note、news |
| `trusted-media` | 可信媒體或第三方深度報導 |
| `rumor` | 傳聞、社群截圖、未確認來源 |

Validator 規則：

- 沒有 URL 的候選不能通過。
- `confidence < 0.75` 必須標記 `needs-human-review`。
- `trusted-media` 的成熟度不得高於 4。
- `rumor` 的成熟度不得高於 2。
- `totalScore >= 80` 且 evidence 不是官方來源時，必須人工審閱。
- Agent 產物永遠只進入 `scoring-review/`，不得直接覆蓋 `src/data/skills.ts`。
